## 四、数据流全链路

```
Bilibili XML 文件
    │
    ▼
BilibiliXmlParser.parse() → List<DanmakuItem>
    │
    ▼
DanmakuRepository.load() → DanmakuTimeIndex (1000ms buckets)
    │
    ▼
MpvClockBridge (Choreographer 每帧回调)
    │
    ├─ PlayerClockProvider.currentPositionMs()  ← MpvPlayerManager
    │
    ▼
DanmakuTimelineEngine.onClockTick()
    │
    ├─ Repository.query() → 时间窗口内弹幕
    ├─ TrackAllocator.allocateScroll/Fixed() → 分配轨道
    ├─ ActiveDanmakuStore → 维护活动弹幕
    │
    ▼
engine.snapshot() → List<DanmakuRenderItem>
    │
    ▼
CanvasRenderer.render()
    │
    ├─ GlyphCache.draw() → 预烘焙 Bitmap
    ├─ lockCanvas / drawBitmap / unlockCanvas
    │
    ▼
SimpleOverlayHost (透明 SurfaceView) → 显示在视频之上
```

## 五、与底板的集成点

// mpv 就绪后创建弹幕控制器
danmakuController = DanmakuControllerImpl(playerManager, playerManager)

fun attachDanmakuOverlay(host: DanmakuOverlayHost) {
    danmakuController.attach(host)
}

// 加载弹幕文件
fun loadDanmaku(path: String) {
    controller.setSource(DanmakuDataSource.FilePath(path))
    controller.load()
}

// 弹幕开关
fun toggleDanmaku() {
    danmakuController.setVisibility(visible)
}

### 已修复的问题

1. **包结构对齐 8.4 节**：桥接接口从 `com.danmakuplayer.bridge` 移至 `com.danmaku.flow.bridge.api`，消除反向依赖
2. **弹幕定位错误**：引擎屏幕尺寸改为每帧从实际 Canvas 动态获取，而非初始化时缓存
3. **弹幕加载概率失败**：增加 Surface 就绪回调，Surface 未创建时延迟重试加载
4. **弹幕数量少**：移除 P0 阶段同屏弹幕数量限制
5. **GlyphCache 性能**：Paint 对象改为类级别复用，避免每帧每条弹幕 new Paint
